5  BLISS-RS Formulation

5.1 Motivation: From Gradients to Surrogates

BLISS-98 couples the local and system levels through sensitivity derivatives \(\partial X^*_{opt} / \partial Z\). Computing these derivatives requires:

  1. A converged system analysis (expensive).
  2. A local optimization at each BBs (moderately expensive).
  3. A sensitivity analysis — typically finite differences or adjoint solves — at the local optimum (often the most expensive step).

These three requirements limit the practical applicability of BLISS-98 in two ways. First, the sensitivity analysis must be repeated each BLISS cycle, making the method computationally intensive for BBs with many variables. Second, because the coupling relies on a local linear approximation, the algorithm can be sensitive to the accuracy of the linearization and may stall near saddle points.

BLISS-RS addresses both limitations by replacing the gradient-based coupling with quadratic response surface surrogates [1]. The key observations motivating this change are:

  • A response surface efficiently explores the entire design space region by evaluating the BB at many sample points rather than linearizing around a single point.
  • Because BBs in BLISS-RS are optimized off-line for each DOE point independently, those optimizations can proceed in strict parallel — the algorithm becomes a workfront that saturates available computing resources.
  • The RS coupling eliminates the need for a system sensitivity analysis. The system optimizer reads coupling information directly from the polynomial coefficients.

5.2 Local Optimization Formulation

In BLISS-RS, each black box \(k\) defines a synthetic objective function that aggregates all of its outputs into a single scalar using weighting factors \(w\). For given fixed values of \(Z\), \(Y^*\), and \(w\), the \(k\)-th BB solves:

\[ \text{Given:} \quad Z, \; Y^*, \; w \tag{5.1}\]

\[ \text{Find:} \quad X \;\text{(local to BB}_k\text{)} \tag{5.2}\]

\[ \text{Minimize:} \quad F_k = \sum_i w_i \, Y^A_i \tag{5.3}\]

\[ \text{Satisfy:} \quad \{h\} = 0, \quad \{g\} \leq 0, \quad X_L \leq X \leq X_U \tag{5.4}\]

The term synthetic is used because the objective \(F_k\) does not represent the actual system objective (range, for example). Instead, it is a proxy that drives each BB toward outputs that are beneficial to the system. The weighting factors \(w\) encode the relative importance of each BB output to the overall system objective.

Note

Weighting factor interpretation. In the supersonic business jet context, the structures BB outputs total weight \(W_T\) and aeroelastic twist \(\Omega\). Both affect range through the Breguet equation. The weight assigned to each (\(w_1\) for \(W_T\), \(w_2\) for \(\Omega\)) indicates how strongly minimizing that output will benefit the system objective. The system optimizer determines the appropriate \(w\) values automatically as part of its decision variables.

The synthetic objective is evaluated at many points \((Z, Y^*, w)\) by running the local optimization for each DOE sample. The collection of optimal values \(F_k^*(Z, Y^*, w)\) forms the data cloud from which the RS is fitted.

5.3 System Optimization Formulation

After the RS models are available, the system optimizer solves:

\[ \text{Given:} \quad \hat{\boldsymbol{\beta}}_k \;\text{for each BB } k \tag{5.5}\]

\[ \text{Find:} \quad Z, \; Y^*, \; w \tag{5.6}\]

\[ \text{Maximize:} \quad F = \text{Range}(Z, Y^*, w) \;\text{via RS models} \tag{5.7}\]

\[ \text{Satisfy:} \quad \{c\} = Y^A - Y^* = 0, \quad \{g\} \leq 0, \quad \text{side constraints} \tag{5.8}\]

where \(\hat{\boldsymbol{\beta}}_k\) are the regression coefficient vectors for the \(k\)-th BB’s response surface model. The coupling constraints \(\{c\} = 0\) ensure that the value of each coupling variable proposed by the system optimizer (\(Y^*\)) matches the value that the corresponding BB would actually produce (\(Y^A\)) given those inputs. In Figure 5.1, coupling constraints are represented as circles where data flow channels from different BBs meet.

5.4 Weighting Factors as System Degrees of Freedom

One of the most important conceptual shifts in BLISS-RS relative to BLISS-98 is the treatment of weighting factors \(w\). In BLISS-98, the weighting factors that define each BB’s composite objective function are held fixed — they are determined a priori by the analyst based on engineering judgement about the relative importance of each output.

In BLISS-RS, the weighting factors \(w\) are system-level decision variables, optimized alongside \(Z\) and \(Y^*\) by the system optimizer. This means the system optimizer can autonomously determine how much each BB should favour, for example, minimizing weight versus maximizing lift-to-drag.

This autonomy comes at a cost: the design space for the system optimizer grows to include \(|Z| + |Y^*| + \sum_k |w_k|\) variables, which increases the complexity of the RS fitting. However, because the system RS is evaluated analytically (not through expensive BB calls), this additional complexity is acceptable.

5.6 Comparison: BLISS-98 vs. BLISS-RS

The central tradeoff is that BLISS-RS requires many more BB evaluations per cycle (one per DOE point per BB) compared to BLISS-98 (one per BB). However, because those evaluations are fully concurrent and because the sensitivity analysis step is eliminated, BLISS-RS is expected to achieve better wall-clock performance on multi-processor systems.

## Comparison: BLISS-98 vs. BLISS-RS

| Feature | BLISS-98 | BLISS-RS |
|---------|----------|---------|
| BB-to-system coupling | Sensitivity derivatives | Quadratic RS surrogates |
| System analysis required per cycle | Yes | No (RS serves this role) |
| Local optimizations per cycle | 1 per BB | $N_S$ per BB (parallelisable) |
| Gradient computation required | Yes | No |
| Handles non-differentiable BBs | No | Yes |
| Prone to local optima | More so | Less so |
| Weighting factors $w$ | Fixed by analyst | Optimised by system level |
| Primary computational cost | Sensitivity analysis | RS generation |

eliminated, BLISS-RS is expected to achieve better wall-clock performance on
