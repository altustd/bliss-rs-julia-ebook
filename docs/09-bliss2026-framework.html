11.1 Why Quadratic Polynomials Are Not Enough

The BLISS-RS surrogate is a second-order polynomial fitted by ordinary least squares. This choice was sensible in 2002: quadratic models are cheap to fit, analytically differentiable, and require at most \(N_S = (n^2+3n+2)/2\) evaluations. For the supersonic business jet’s aerodynamics black box (\(n = 14\), \(N_S = 120\)) the fit is tractable and the gradient is free.

But quadratic polynomials make a strong structural assumption: the response is smooth, unimodal, and well-approximated by a paraboloid over the entire search interval. When the interval is large relative to the response variation this assumption fails — the polynomial interpolates through sample points but extrapolates poorly. Worse, it provides no indication of where it is unreliable.

Figure 11.1 illustrates the difference on a one-dimensional slice through the supersonic business jet range function.

Two problems are visible in Figure 11.1:

  1. The quadratic polynomial misses the local maximum near 55° and predicts negative range at the high-sweep extreme — physically impossible extrapolation.
  2. It gives no warning that its prediction is unreliable. The GP correctly widens its uncertainty bands in the data-sparse high-sweep region.

11.2 Gaussian Process Surrogates

A Gaussian Process is a probability distribution over functions. Any finite collection of function values follows a multivariate normal distribution:

\[ f(\mathbf{x}_1), \ldots, f(\mathbf{x}_n) \sim \mathcal{N}(\mathbf{m}, K) \tag{11.1}\]

where \(m_i = m(\mathbf{x}_i)\) is the prior mean (typically zero after output normalisation) and \(K_{ij} = k(\mathbf{x}_i, \mathbf{x}_j)\) is the kernel (covariance) function encoding prior beliefs about function smoothness.

11.2.1 Kernel Functions

Three kernels are implemented in src/bliss_gp.jl:

Squared Exponential (SE) — infinitely differentiable, very smooth: \[ k_{SE}(\mathbf{x}, \mathbf{x}') = \sigma^2 \exp\!\left(-\frac{\|\mathbf{x}-\mathbf{x}'\|^2}{2\ell^2}\right) \tag{11.2}\]

Matérn 5/2 — twice differentiable, preferred for physical engineering responses: \[ k_{M52}(r) = \sigma^2\!\left(1 + \frac{\sqrt{5}\,r}{\ell} + \frac{5r^2}{3\ell^2}\right)\exp\!\left(-\frac{\sqrt{5}\,r}{\ell}\right) \tag{11.3}\]

ARD-SE — independent length scale \(\ell_i\) per dimension, allowing automatic relevance determination: unimportant variables get large \(\ell_i\) and are effectively ignored by the surrogate.

11.2.2 Posterior Inference

Given \(n\) noisy observations \(\mathbf{y} = f(\mathbf{X}) + \boldsymbol{\varepsilon}\) with \(\varepsilon \sim \mathcal{N}(0, \sigma_n^2 I)\), the GP posterior at test points \(\mathbf{X}_*\) is:

\[ \mu(\mathbf{X}_*) = k(\mathbf{X}_*, \mathbf{X})\,(K + \sigma_n^2 I)^{-1}\,\mathbf{y} \tag{11.4}\]

\[ \sigma^2(\mathbf{X}_*) = k(\mathbf{X}_*, \mathbf{X}_*) - k(\mathbf{X}_*, \mathbf{X})\,(K + \sigma_n^2 I)^{-1}\,k(\mathbf{X}, \mathbf{X}_*) \tag{11.5}\]

In code this is computed efficiently via a Cholesky factorisation \(K + \sigma_n^2 I = LL^T\):

\[ \boldsymbol{\alpha} = L^{-T}(L^{-1}\mathbf{y}), \qquad \mu_* = k_*^T\boldsymbol{\alpha}, \qquad \sigma^2_* = k_{**} - \|L^{-1}k_*\|^2 \tag{11.6}\]

11.2.3 Hyperparameter Optimisation via Marginal Likelihood

The length scale \(\ell\) and signal variance \(\sigma^2\) are not free parameters to be tuned by hand — they are inferred from the data itself by maximising the log marginal likelihood:

\[ \log p(\mathbf{y} \mid \mathbf{X}, \theta) = -\tfrac{1}{2}\mathbf{y}^T K_\theta^{-1}\mathbf{y} -\tfrac{1}{2}\log|K_\theta| -\tfrac{n}{2}\log 2\pi \tag{11.7}\]

The three terms have intuitive interpretations: the first penalises poor data fit, the second penalises model complexity (larger \(\ell\) = simpler model = smaller \(|K|\)), and the third is a normalisation constant. Optimising Equation 11.7 over \(\log\ell\) and \(\log\sigma^2\) with L-BFGS typically finds a good hyperparameter set in under 300 gradient evaluations.

11.3 Expected Improvement Acquisition

At each Bayesian optimisation step we must choose where to evaluate the black box next. The Expected Improvement (EI) acquisition function provides a principled answer: evaluate at the point that maximally improves on the current best, in expectation over the GP posterior.

For maximisation with current best \(f^* = \max_i y_i\):

\[ \text{EI}(\mathbf{x}) = \mathbb{E}\!\left[\max(f(\mathbf{x}) - f^*, 0)\right] = (m(\mathbf{x}) - f^* - \xi)\,\Phi(z) + \sigma(\mathbf{x})\,\phi(z) \tag{11.8}\]

\[ z = \frac{m(\mathbf{x}) - f^* - \xi}{\sigma(\mathbf{x})} \tag{11.9}\]

where \(\Phi\) and \(\phi\) are the standard normal CDF and PDF, and \(\xi \geq 0\) is an exploration bonus. When \(\sigma(\mathbf{x}) = 0\) (at an observed point) EI is zero — the algorithm never revisits the same point.

Note

EI vs. Nelder-Mead over RS. BLISS-RS uses Nelder-Mead to minimise \(-\hat{f}(\mathbf{x})\) where \(\hat{f}\) is the polynomial surface. This exploits the current best estimate but ignores uncertainty: it will avoid high-uncertainty regions even if they might contain the global optimum. EI naturally balances exploitation (high \(m\)) against exploration (high \(\sigma\)).

Code 
f_best = maximum(y_tr)
y_n2   = (y_tr .- mean(y_tr)) ./ (std(y_tr) + 1e-8)
gp_n2  = fit_gp(reshape(x_c, 10, 1), y_n2, gp_n.kernel; σ_n=1e-3)
μ_n2, σ²_n2 = predict_gp(gp_n2, reshape(x_tc, 200, 1))
μ_sc   = μ_n2 .* std(y_tr) .+ mean(y_tr)
σ_sc   = sqrt.(σ²_n2) .* std(y_tr)
ei_vals = expected_improvement(μ_sc, σ_sc.^2, f_best; ξ=10.0)

p1 = plot(sweep_range, μ_sc, lw=2, color=:steelblue, label="GP mean",
          ylabel="Range (NM)", xlabel="", title="GP Posterior",
          legend=:topleft)
plot!(sweep_range, μ_sc .- 2σ_sc, fillrange=μ_sc .+ 2σ_sc,
      fillalpha=0.18, color=:steelblue, lw=0, label="±2σ")
scatter!(x_tr, y_tr, color=:black, markersize=5, label="Obs.")
vline!([sweep_range[argmax(ei_vals)]], color=:crimson, lw=2, linestyle=:dash, label="Next EI")

p2 = plot(sweep_range, ei_vals, lw=2, color=:darkorange,
          fill=(0, 0.25, :darkorange), ylabel="EI", xlabel="Wing sweep Λ_w (deg)",

11.4 Latin Hypercube Sampling

BLISS-2026 replaces D-optimal designs with Latin Hypercube Sampling (LHS) for the initial DOE. LHS divides each variable dimension into \(n\) equal-probability strata and places exactly one sample in each stratum, ensuring good marginal coverage with no clustering.

11.5 The BLISS-2026 Algorithm

The complete algorithm (implemented in src/bliss_gp.jl) is:

The formal statement of the BLISS-2026 system optimisation at each outer iteration is:

\[ \text{Find:} \quad \mathbf{Z}^* = \arg\max_{\mathbf{Z} \in \mathcal{D}_k} \text{EI}(\mathbf{Z}; \hat{f}_{GP}, f^*_k, \xi) \tag{11.10}\]

where \(\mathcal{D}_k\) is the trust-region at iteration \(k\), \(\hat{f}_{GP}\) is the current GP posterior, and \(f^*_k\) is the best observed objective so far. The outer iterations progressively refine \(\mathcal{D}_k\) via the trust-region update.

11.8 Convergence Comparison: BLISS-RS vs. BLISS-2026

Code 
# Run BLISS-RS (from ch07 implementation)
using Optim

function run_blissrs_ssbj(n_iter=8, K=0.8; seed=42)
    bounds_ = copy(bounds_ssbj)
    Z_hist  = Vector{Vector{Float64}}()
    obj_hist = Float64[]
    rng = MersenneTwister(seed)

    for it in 1:n_iter
        n_pts = max(min_rs_points(n_Z_ssbj) + 5, 55)
        pts_c = d_optimal_points(n_Z_ssbj, n_pts; seed=seed+it)
        pts_n = similar(pts_c)
        for j in 1:n_Z_ssbj
            lo, hi = bounds_[j,1], bounds_[j,2]
            pts_n[:,j] = lo .+ pts_c[:,j] .* (hi-lo)
        end
        y_vals = map(1:n_pts) do i
            try ssbj_bb(pts_n[i,:]) catch; 0.0 end
        end
        b = fit_rs(pts_c, y_vals)
        neg_rs(ξ) = -only(predict_rs(reshape(ξ,1,:), b))
        res = Optim.optimize(neg_rs, fill(0.5, n_Z_ssbj), NelderMead(),
                             Optim.Options(iterations=2000, show_trace=false))
        ξ_opt = clamp.(Optim.minimizer(res), 0.0, 1.0)
        Z_opt = [bounds_[j,1] + ξ_opt[j]*(bounds_[j,2]-bounds_[j,1]) for j in 1:n_Z_ssbj]
        obj   = try ssbj_bb(Z_opt) catch; 0.0 end
        push!(Z_hist, Z_opt)
        push!(obj_hist, obj)
        bounds_ = update_intervals(ξ_opt, bounds_; K=K)
    end
    return Z_hist, obj_hist
end

_, obj_rs = run_blissrs_ssbj(8, 0.8; seed=42)
obj_gp = result_gp.obj_history

11.9 GP Uncertainty as a Design Tool

The GP posterior provides something the polynomial surrogate cannot: a calibrated probability distribution over function values at any unsampled point. This has immediate practical value in MDO:

  1. Stopping criterion: When \(\max_Z \sigma(Z) < \epsilon\) across the current search interval, the GP is confident everywhere — no further sampling is needed.
  2. Risk-aware optimisation: Instead of maximising \(\mu(\mathbf{Z})\), maximise \(\mu(\mathbf{Z}) - k\sigma(\mathbf{Z})\) to find designs that are good and certain.
  3. Sensitivity analysis: The GP length scales \(\ell_i\) from an ARD kernel directly indicate variable importance — large \(\ell_i\) means the objective is insensitive to variable \(i\).

Z_opt_final = result_gp.Z_history[end]

## Why Quadratic Polynomials Are Not Enough

The BLISS-RS surrogate is a second-order polynomial fitted by ordinary least squares.
This choice was sensible in 2002: quadratic models are cheap to fit, analytically
differentiable, and require at most $N_S = (n^2+3n+2)/2$ evaluations. For the
supersonic business jet's aerodynamics black box ($n = 14$, $N_S = 120$) the fit is
tractable and the gradient is free.

But quadratic polynomials make a strong structural assumption: the response is smooth,
unimodal, and well-approximated by a paraboloid over the entire search interval. When
the interval is large relative to the response variation this assumption fails — the
polynomial interpolates through sample points but extrapolates poorly. Worse, it
provides no indication of *where* it is unreliable.

@fig-poly-vs-gp illustrates the difference on a one-dimensional slice through the
supersonic business jet range function.

## Gaussian Process Surrogates

A **Gaussian Process** is a probability distribution over functions. Any finite
collection of function values follows a multivariate normal distribution:

$$
f(\mathbf{x}_1), \ldots, f(\mathbf{x}_n) \sim \mathcal{N}(\mathbf{m}, K)
$$ {#eq-gp-def}

where $m_i = m(\mathbf{x}_i)$ is the prior mean (typically zero after output
normalisation) and $K_{ij} = k(\mathbf{x}_i, \mathbf{x}_j)$ is the **kernel
(covariance) function** encoding prior beliefs about function smoothness.

### Kernel Functions

Three kernels are implemented in `src/bliss_gp.jl`:

**Squared Exponential (SE)** — infinitely differentiable, very smooth:
$$
k_{SE}(\mathbf{x}, \mathbf{x}') = \sigma^2 \exp\!\left(-\frac{\|\mathbf{x}-\mathbf{x}'\|^2}{2\ell^2}\right)
$$ {#eq-se-kernel}

**Matérn 5/2** — twice differentiable, preferred for physical engineering responses:
$$
k_{M52}(r) = \sigma^2\!\left(1 + \frac{\sqrt{5}\,r}{\ell} + \frac{5r^2}{3\ell^2}\right)\exp\!\left(-\frac{\sqrt{5}\,r}{\ell}\right)
$$ {#eq-matern52}

**ARD-SE** — independent length scale $\ell_i$ per dimension, allowing automatic
relevance determination: unimportant variables get large $\ell_i$ and are
effectively ignored by the surrogate.

### Posterior Inference

Given $n$ noisy observations $\mathbf{y} = f(\mathbf{X}) + \boldsymbol{\varepsilon}$
with $\varepsilon \sim \mathcal{N}(0, \sigma_n^2 I)$, the GP posterior at test points
$\mathbf{X}_*$ is:

$$
\mu(\mathbf{X}_*) = k(\mathbf{X}_*, \mathbf{X})\,(K + \sigma_n^2 I)^{-1}\,\mathbf{y}
$$ {#eq-gp-mean}

$$
\sigma^2(\mathbf{X}_*) = k(\mathbf{X}_*, \mathbf{X}_*) - k(\mathbf{X}_*, \mathbf{X})\,(K + \sigma_n^2 I)^{-1}\,k(\mathbf{X}, \mathbf{X}_*)
$$ {#eq-gp-var}

In code this is computed efficiently via a Cholesky factorisation $K + \sigma_n^2 I = LL^T$:

$$
\boldsymbol{\alpha} = L^{-T}(L^{-1}\mathbf{y}), \qquad
\mu_* = k_*^T\boldsymbol{\alpha}, \qquad
\sigma^2_* = k_{**} - \|L^{-1}k_*\|^2
$$ {#eq-cholesky-inference}

### Hyperparameter Optimisation via Marginal Likelihood

The length scale $\ell$ and signal variance $\sigma^2$ are not free parameters to
be tuned by hand — they are inferred from the data itself by maximising the **log
marginal likelihood**:

$$
\log p(\mathbf{y} \mid \mathbf{X}, \theta) =
  -\tfrac{1}{2}\mathbf{y}^T K_\theta^{-1}\mathbf{y}
  -\tfrac{1}{2}\log|K_\theta|
  -\tfrac{n}{2}\log 2\pi
$$ {#eq-lml}

The three terms have intuitive interpretations: the first penalises poor data fit,
the second penalises model complexity (larger $\ell$ = simpler model = smaller $|K|$),
and the third is a normalisation constant. Optimising @eq-lml over $\log\ell$ and
$\log\sigma^2$ with L-BFGS typically finds a good hyperparameter set in under 300
gradient evaluations.

## Expected Improvement Acquisition

At each Bayesian optimisation step we must choose where to evaluate the black box
next. The **Expected Improvement** (EI) acquisition function provides a principled
answer: evaluate at the point that maximally improves on the current best, in
expectation over the GP posterior.

For **maximisation** with current best $f^* = \max_i y_i$:

$$
\text{EI}(\mathbf{x}) = \mathbb{E}\!\left[\max(f(\mathbf{x}) - f^*, 0)\right]
= (m(\mathbf{x}) - f^* - \xi)\,\Phi(z) + \sigma(\mathbf{x})\,\phi(z)
$$ {#eq-ei}

$$
z = \frac{m(\mathbf{x}) - f^* - \xi}{\sigma(\mathbf{x})}
$$ {#eq-ei-z}

where $\Phi$ and $\phi$ are the standard normal CDF and PDF, and $\xi \geq 0$ is
an exploration bonus. When $\sigma(\mathbf{x}) = 0$ (at an observed point) EI is
zero — the algorithm never revisits the same point.

::: {.callout-note}
**EI vs. Nelder-Mead over RS.** BLISS-RS uses Nelder-Mead to minimise
$-\hat{f}(\mathbf{x})$ where $\hat{f}$ is the polynomial surface. This exploits
the current best estimate but ignores uncertainty: it will avoid high-uncertainty
regions even if they might contain the global optimum. EI naturally balances
exploitation (high $m$) against exploration (high $\sigma$).
:::

## Latin Hypercube Sampling

BLISS-2026 replaces D-optimal designs with **Latin Hypercube Sampling** (LHS) for
the initial DOE. LHS divides each variable dimension into $n$ equal-probability
strata and places exactly one sample in each stratum, ensuring good marginal coverage
with no clustering.

## The BLISS-2026 Algorithm

The complete algorithm (implemented in `src/bliss_gp.jl`) is:


The formal statement of the BLISS-2026 system optimisation at each outer iteration
is:

$$
\text{Find:} \quad \mathbf{Z}^* = \arg\max_{\mathbf{Z} \in \mathcal{D}_k}
  \text{EI}(\mathbf{Z}; \hat{f}_{GP}, f^*_k, \xi)
$$ {#eq-bliss2026-obj}

where $\mathcal{D}_k$ is the trust-region at iteration $k$, $\hat{f}_{GP}$ is the
current GP posterior, and $f^*_k$ is the best observed objective so far. The
outer iterations progressively refine $\mathcal{D}_k$ via the trust-region update.

## Convergence Comparison: BLISS-RS vs. BLISS-2026

```{julia}
#| label: fig-comparison-convergence
#| fig-cap: "Convergence comparison: BLISS-RS (quadratic RS + Nelder-Mead) vs. BLISS-2026 (GP + EI) on the supersonic business jet. BLISS-2026 reaches a higher objective value and shows smoother monotone improvement — the EI acquisition prevents wasted evaluations on already-explored regions."
#| code-fold: true

# Run BLISS-RS (from ch07 implementation)
using Optim

function run_blissrs_ssbj(n_iter=8, K=0.8; seed=42)
    bounds_ = copy(bounds_ssbj)
    Z_hist  = Vector{Vector{Float64}}()
    obj_hist = Float64[]
    rng = MersenneTwister(seed)

    for it in 1:n_iter
        n_pts = max(min_rs_points(n_Z_ssbj) + 5, 55)
        pts_c = d_optimal_points(n_Z_ssbj, n_pts; seed=seed+it)
        pts_n = similar(pts_c)
        for j in 1:n_Z_ssbj
            lo, hi = bounds_[j,1], bounds_[j,2]
            pts_n[:,j] = lo .+ pts_c[:,j] .* (hi-lo)
        end
        y_vals = map(1:n_pts) do i
            try ssbj_bb(pts_n[i,:]) catch; 0.0 end
        end
        b = fit_rs(pts_c, y_vals)
        neg_rs(ξ) = -only(predict_rs(reshape(ξ,1,:), b))
        res = Optim.optimize(neg_rs, fill(0.5, n_Z_ssbj), NelderMead(),
                             Optim.Options(iterations=2000, show_trace=false))
        ξ_opt = clamp.(Optim.minimizer(res), 0.0, 1.0)
        Z_opt = [bounds_[j,1] + ξ_opt[j]*(bounds_[j,2]-bounds_[j,1]) for j in 1:n_Z_ssbj]
        obj   = try ssbj_bb(Z_opt) catch; 0.0 end
        push!(Z_hist, Z_opt)
        push!(obj_hist, obj)
        bounds_ = update_intervals(ξ_opt, bounds_; K=K)
    end
    return Z_hist, obj_hist
end
```

## GP Uncertainty as a Design Tool

The GP posterior provides something the polynomial surrogate cannot: a calibrated
probability distribution over function values at any unsampled point. This has
immediate practical value in MDO:

1. **Stopping criterion**: When $\max_Z \sigma(Z) < \epsilon$ across the current
   search interval, the GP is confident everywhere — no further sampling is needed.
2. **Risk-aware optimisation**: Instead of maximising $\mu(\mathbf{Z})$, maximise
   $\mu(\mathbf{Z}) - k\sigma(\mathbf{Z})$ to find designs that are good *and*
   certain.
3. **Sensitivity analysis**: The GP length scales $\ell_i$ from an ARD kernel
   directly indicate variable importance — large $\ell_i$ means the objective
   is insensitive to variable $i$.
