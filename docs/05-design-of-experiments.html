6.1 Context for Computer Experiments

In classical physical experiments, Design of Experiments (DOE) strategies minimize the variance of the estimated regression coefficients in the presence of random measurement noise [1]. Common designs such as the Central Composite Design (CCD) and the Box-Behnken design were developed with this goal in mind.

Computer experiments present a fundamentally different setting [2]:

  • Outputs are deterministic — the same inputs always produce the same output. Replication provides no statistical benefit.
  • The primary concern is space-filling — ensuring sample points cover the design space well so that the response surface captures the underlying function accurately throughout the region.
  • Cost is per-evaluation, not per-experiment-run, so economizing on total sample count is important.

Altus (2002) investigated three point placement schemes for BLISS-RS:

  1. Hypersphere point placement — points dispersed uniformly on the surface of an \(n\)-dimensional unit hypersphere and projected inward.
  2. Extreme (random) point placement — uniform random samples in the coded design space, serving as a baseline comparison.
  3. D-optimal design — points that maximize \(\det([X]^T[X])\), chosen via a coordinate exchange algorithm.

The thesis concluded that D-optimal placement provided the best balance of accuracy and economy for BLISS-RS, and that was the scheme adopted for the supersonic business jet results.

6.2 Hypersphere Point Placement

The hypersphere scheme attempts to distribute points uniformly across the design space by placing them on the surface of a unit hypersphere in \(n\) dimensions, then projecting inward. The anti-bunching property of the sphere surface — the angular repulsion between points — promotes dispersion.

A point is generated by sampling from a standard Gaussian distribution in \(n\) dimensions and normalising to unit length:

\[ \mathbf{u} \sim \mathcal{N}(0, I_n), \qquad \hat{\mathbf{u}} = \frac{\mathbf{u}}{\|\mathbf{u}\|} \tag{6.1}\]

The full algorithm is detailed in Appendix A. An important characteristic is that the number of points is limited only by the minimum required for the RS model — unlike CCD or Box-Behnken designs, whose point counts grow as specific functions of \(n\).

6.3 D-Optimal Point Placement

The D-optimal design criterion seeks to maximize the determinant of the information matrix \([X]^T[X]\):

\[ \text{Maximize} \quad \det\!\left([X(\xi)]^T[X(\xi)]\right) \tag{6.2}\]

This is equivalent to minimizing the volume of the confidence ellipsoid for the regression coefficients \(\hat{\boldsymbol{\beta}}\). A design with a high \(\det([X]^T[X])\) estimates the coefficients with minimum overall uncertainty.

D-optimal designs are generated algorithmically. The coordinate exchange algorithm (Cook & Nachtsheim, 1980) works as follows:

  1. Initialize with a random set of candidate points \(\xi_1, \ldots, \xi_m\).
  2. For each point \(\xi_i\), for each coordinate \(j\):
    • Try replacing \(\xi_{i,j}\) with a new candidate value.
    • Accept if \(\det([X]^T[X])\) increases.
  3. Repeat until no improvement is found.

6.3.1 Key Features of D-Optimal Designs

One particularly valuable feature of the D-optimal algorithm is that the user can specify the exact number of points desired (regardless of the classic design formulas). This is critical in BLISS-RS because:

  • The minimum \(N_S\) from Equation 5.9 can be augmented by any number of additional points to improve fit robustness, without the rigidity of CCD or Box-Behnken counts.
  • In each BLISS iteration, if a particular sample point fails to converge in its local BB optimization, a replacement random point can be substituted without invalidating the design.
Code 
n_vars = 3
ns_min = min_rs_points(n_vars)
n_pts_range = ns_min:2:(ns_min + 30)

log_det_dopt = Float64[]
log_det_hs   = Float64[]

for n_pts in n_pts_range
    pts_d = d_optimal_points(n_vars, n_pts; seed=42)
    pts_h = hypersphere_points(n_vars, n_pts; seed=42)
    X_d = rs_design_matrix(pts_d);  push!(log_det_dopt, log(max(abs(det(X_d'*X_d)), 1e-300)))
    X_h = rs_design_matrix(pts_h);  push!(log_det_hs,   log(max(abs(det(X_h'*X_h)), 1e-300)))
end

plot(collect(n_pts_range), log_det_dopt, lw=2, color=:darkorange,
     label="D-Optimal",
     xlabel="Sample size NS", ylabel=L"\log\,\det(X^TX)",
     title="Information Matrix Det. vs. Sample Size (n=3)",
     legend=:bottomright, grid=true, gridalpha=0.3, marker=:circle, markersize=4)
plot!(collect(n_pts_range), log_det_hs, lw=2, color=:steelblue,
     label="Hypersphere", marker=:square, markersize=4)
vline!([ns_min], color=:black, linestyle=:dash, lw=1.2, label="NS_min = $ns_min")

6.4 Coded and Natural Variables

DOE algorithms work in a coded design space where all variables are normalised to the interval \([-1, 1]\) (or equivalently \([0, 1]\)). Physical design variables, by contrast, occupy widely different scales and units (degrees, square feet, unitless ratios). A coordinate mapping connects the two:

\[ \xi = \frac{X - X_C}{\Delta X / 2} = \frac{2(X - X_L)}{X_U - X_L} - 1 \tag{6.3}\]

where \(X_L\) and \(X_U\) are the lower and upper bounds on the physical variable \(X\), \(X_C = (X_L + X_U)/2\) is the centre, and \(\xi \in [-1, 1]\) is the coded variable. Conversely:

\[ X = X_L + \frac{\xi + 1}{2}(X_U - X_L) \tag{6.4}\]

Code 
# Demonstrate the mapping for wing sweep angle (40° to 70°)
x_L, x_H = 40.0, 70.0
coded_vals   = [-1.0, -0.5, 0.0, 0.5, 1.0]
natural_vals = code_to_natural.(coded_vals, x_L, x_H)

println("Wing Sweep Mapping (xL=40°, xH=70°)")
println("ξ (coded)  →  Λ_w (natural, deg)")
for (ξ, x) in zip(coded_vals, natural_vals)
    @printf("  %+5.2f   →  %6.2f°\n", ξ, x)
end
Wing Sweep Mapping (xL=40°, xH=70°)
ξ (coded)  →  Λ_w (natural, deg)
  -1.00   →   40.00°
  -0.50   →   47.50°
  +0.00   →   55.00°
  +0.50   →   62.50°
  +1.00   →   70.00°

In BLISS-RS, the bounds \(X_L\) and \(X_U\) are adaptive — they start as the physical design limits and are reduced each iteration to focus the RS on a smaller, more reliably-approximated region around the current system optimum (Chapter 6).

6.5 Space-Filling Diagnostics

Two useful diagnostics for assessing point placement quality are:

Minimum distance (\(d_{\min}\)) — the smallest Euclidean distance between any pair of sample points. A high \(d_{\min}\) indicates no two points are excessively close.

Coverage — the maximum distance from any point in the design space to its nearest sample point. Low coverage means the design has explored the space thoroughly.

Code 
function min_pairwise_dist(pts::Matrix{Float64})
    m = size(pts, 1)
    d_min = Inf
    for i in 1:m, j in (i+1):m
        d = norm(pts[i,:] - pts[j,:])
        d_min = min(d_min, d)
    end
    return d_min
end

n_trials = 200
d_random  = [min_pairwise_dist(random_points(3, 20; seed=seed)) for seed in 1:n_trials]
d_dopt    = [min_pairwise_dist(d_optimal_points(3, 20; seed=seed, n_iter=300)) for seed in 1:n_trials]

stephist(d_random, bins=20, label="Random", color=:crimson, alpha=0.6,
         xlabel="Minimum pairwise distance", ylabel="Count",
         title="Space-Filling Quality (n=3, NS=20, 200 trials)")
stephist!(d_dopt, bins=20, label="D-Optimal", color=:steelblue, alpha=0.6)

The D-optimal algorithm reliably produces designs with larger minimum pairwise distances, avoiding the occasional clustering that plagues pure random placement. This is the principal reason the D-optimal design was selected for the BLISS-RS results in Chapter 7.

## D-Optimal Point Placement

The **D-optimal design** criterion seeks to maximize the determinant of the
information matrix $[X]^T[X]$:

$$
\text{Maximize} \quad \det\!\left([X(\xi)]^T[X(\xi)]\right)
$$ {#eq-d-optimal}

This is equivalent to minimizing the volume of the confidence ellipsoid for the
regression coefficients $\hat{\boldsymbol{\beta}}$. A design with a high
$\det([X]^T[X])$ estimates the coefficients with minimum overall uncertainty.

D-optimal designs are generated algorithmically. The **coordinate exchange**
algorithm (Cook & Nachtsheim, 1980) works as follows:

1. Initialize with a random set of candidate points $\xi_1, \ldots, \xi_m$.
2. For each point $\xi_i$, for each coordinate $j$:
   - Try replacing $\xi_{i,j}$ with a new candidate value.
   - Accept if $\det([X]^T[X])$ increases.
3. Repeat until no improvement is found.
