6.1 Overview

The goal of Response Surface Methodology (RSM) is to construct a mathematical model — a response surface — that approximates the relationship between a set of input factors and an output response [1,2]. In the context of BLISS-RS, the factors are the system-level design variables \(Z\) and coupling variables \(Y^*\), and the response is the locally-optimal BB output \(F_k^*\).

RSM was originally developed for physical experiments where noise is present. When applied to computer experiments (deterministic simulations), there is no measurement error: running the same code twice with the same inputs always produces the same output. This changes how sample points should be placed — classical designs that replicate observations to estimate variance are wasteful; instead, the emphasis shifts to efficient space-filling.

BLISS-RS uses RSM in the computer-experiment sense: each “experiment” is a local black-box optimization, and the response surface is fitted through the resulting optimal outputs.

6.2 Construction of Empirical Models

6.2.1 Linear Regression

In most RSM applications, the true response function is unknown and we seek a polynomial approximation. The simplest choice is a first-order (linear) model with \(n\) independent variables:

\[ Y = \beta_0 + \sum_{i=1}^{n} \beta_i x_i + \varepsilon \tag{6.1}\]

where \(\beta_0, \beta_1, \ldots, \beta_n\) are unknown regression coefficients and \(\varepsilon\) is a random error (zero for deterministic computer experiments).

This is a linear model because it is a linear function of the unknown parameters \(\beta\), not necessarily of the variables \(x_i\). Interaction terms (\(\beta_{12} x_1 x_2\)) and squared terms (\(\beta_{11} x_1^2\)) can be added while maintaining linearity in parameters.

Suppose the response \(Y\) is observed \(m\) times. Each observation \(y_i\) satisfies:

\[ y_i = \beta_0 + \sum_{j=1}^n \beta_j x_{ij} + \varepsilon_i, \quad i = 1, 2, \ldots, m \tag{6.2}\]

In matrix form:

\[ \{y\} = [X]\{\beta\} + \{\varepsilon\} \tag{6.3}\]

where \(\{y\}\) is \(m \times 1\), \([X]\) is an \(m \times p\) design matrix, \(\{\beta\}\) is \(p \times 1\), and \(\{\varepsilon\}\) is \(m \times 1\).

Least squares estimation minimises the sum of squared residuals:

\[ L = \{\varepsilon\}^T\{\varepsilon\} = (\{y\} - [X]\{\beta\})^T(\{y\} - [X]\{\beta\}) \tag{6.4}\]

Setting \(\partial L / \partial \{\beta\} = 0\) yields the normal equations:

\[ [X]^T[X]\{b\} = [X]^T\{y\} \tag{6.5}\]

The least-squares estimator \(\{b\}\) of \(\{\beta\}\) is:

\[ \{b\} = \left([X]^T[X]\right)^{-1}[X]^T\{y\} \tag{6.6}\]

The fitted values are:

\[ \{\hat{y}\} = [X]\{b\} \tag{6.7}\]

6.2.2 Second-Order Models

For BLISS-RS, first-order models are generally insufficient because the BB responses exhibit curvature. A second-order (quadratic) model adds pure quadratic and cross-product (interaction) terms:

\[ Y = \beta_0 + \sum_{i=1}^n \beta_i x_i + \sum_{i=1}^n \beta_{ii} x_i^2 + \sum_{i < j} \beta_{ij} x_i x_j \tag{6.8}\]

The minimum number of observations required to uniquely fit a second-order model in \(n\) variables is:

\[ N_S = \frac{n^2 + 3n + 2}{2} \tag{6.9}\]

Code 
println("| n | p (coefficients) | NS (min. points) | NS / p |")
println("|---|-----------------|-----------------|-------|")
for n in 1:12
    p  = 1 + n + n*(n+1)÷2
    ns = min_rs_points(n)
    println("| $n | $p | $ns | $(round(ns/p, digits=2)) |")
end

Note that \(N_S = p\) — the minimum design is exactly determined, with no degrees of freedom for error estimation. In practice, BLISS-RS uses slightly more than \(N_S\) points (over-determined system) to improve conditioning and fit robustness.

6.2.3 Julia Implementation

With 12 points for a 2-variable model (\(N_S = 6\)), the least-squares fit recovers the true quadratic exactly (within floating-point precision), confirming the implementation.

6.3 Error Analysis of Fitted Response Surfaces

6.3.1 Test Functions

Altus (2002) assessed RS accuracy using two test functions across a range of dimensionalities \(N_X\) and sample sizes \(N_S\) [3]:

Test 1 — Fitting a quadratic RS to a quadratic function. A correctly implemented quadratic model recovers a quadratic function exactly when \(N_S \geq N_S^{\min}\), so this is a code-verification test, not a fidelity test.

Test 2 — Fitting a quadratic RS to the generalised Rosenbrock (Banana) function, which has a narrow curved valley and is substantially non-quadratic:

\[ f(\mathbf{x}) = \sum_{i=1}^{N_X - 1}\left[100(x_{i+1} - x_i^2)^2 + (x_i - 1)^2\right] \tag{6.10}\]

6.3.2 Corrected Error Metrics

The original thesis reported a single point-wise normalized error:

\[ \text{Error}_{\text{Altus}} = \frac{|f_{\text{actual}} - f_{RS}|}{|f_{\text{actual}}|} \tag{6.11}\]

This metric has two deficiencies identified during IV&V:

  1. Singularity — when \(f_{\text{actual}} \to 0\) (e.g. the aeroelastic twist variable \(\Theta\), which can change sign during optimization), the metric diverges. The original MATLAB code masked this with an implicit numerical floor.

  2. Point-wise only — no summary statistic across the design space, no out-of-sample validation, no confidence information.

The corrected validation framework uses three complementary metrics:

\[ \text{RMSE} = \sqrt{\frac{1}{m}\sum_{i=1}^{m}(y_i - \hat{y}_i)^2} \tag{6.12}\]

\[ R^2 = 1 - \frac{\sum(y_i - \hat{y}_i)^2}{\sum(y_i - \bar{y})^2} \tag{6.13}\]

\[ \text{NMAE} = \frac{1}{m}\sum_{i=1}^{m} \frac{|y_i - \hat{y}_i|}{\max(|y_i|,\, \varepsilon \cdot \max_j|y_j|)} \tag{6.14}\]

where \(\varepsilon = 10^{-8}\) provides the stabilizing floor absent from Eq. 16. All three are computed both in-sample (fit quality) and leave-one-out cross-validated (predictive quality). LOOCV is the standard for deterministic computer experiments because there is no measurement noise to justify a held-out test set.

Code 
rosenbrock(x) = sum(100*(x[i+1] - x[i]^2)^2 + (x[i]-1)^2 for i in 1:length(x)-1)

println("| NX | NS | In-sample R² | LOOCV RMSE | LOOCV R² | LOOCV NMAE |")

            nv, ns,

            v.loocv.rmse,

end

function rs_loocv_study(n_vars; seed=42)

    ns_range = ns_min:4:(ns_min + 60)
    rmse_vec = Float64[]
    r2_vec   = Float64[]

        pts = lhs_points(n_vars, ns; seed=seed)
        pts_coded = 2 .* pts .- 1.0

        push!(rmse_vec, loo.rmse)
        push!(r2_vec,   max(loo.r2, -1.0))

end

Figure 6.1 reproduces and extends the accuracy study from Altus (2002) Figs. 13–16 using LOOCV RMSE in place of the point-wise metric. The key finding is unchanged: for low-dimensional BBs (\(N_X \leq 5\)), a quadratic RS achieves acceptable accuracy with moderate over-determination (\(N_S \approx 1.5 N_S^{\min}\)). For high-dimensional BBs (\(N_X = 8\), as in the aerodynamics BB), predictive accuracy remains poor until \(N_S \gg N_S^{\min}\), motivating either additional sample points or a more expressive surrogate (Gaussian Process, Chapter 9).

The contour plots in Figure 6.2 illustrate an important limitation of quadratic surrogates: the actual Rosenbrock valley follows a parabolic curve \(x_2 = x_1^2\), while the quadratic RS can only represent it as an ellipsoidal bowl. This mismatch motivates the interval reduction strategy in Chapter 6 — by shrinking the design space region around the optimum each iteration, subsequent RS models are fitted to smaller, more nearly-quadratic sub-regions where the approximation is more accurate.

6.4 Conditioning and Over-Determination

The normal equations (Equation 6.5) can become ill-conditioned when sample points are clustered or collinear. Good conditioning requires the design matrix \([X]\) to have columns that are mutually uncorrelated — a property promoted by the D-optimal design strategy described in Chapter 5.

Code 
function condition_number_study(n_vars::Int, n_pts_range, n_trials::Int=10; seed=42)
    rng = MersenneTwister(seed)
    cond_random = Float64[]
    cond_dopt   = Float64[]

    for n_pts in n_pts_range
        # Random points
        conds = Float64[]
        for trial in 1:n_trials
            pts = rand(rng, n_pts, n_vars)
            X   = rs_design_matrix(pts)
            push!(conds, cond(X' * X))
        end
        push!(cond_random, median(conds))

        # D-optimal points
        pts_d = d_optimal_points(n_vars, n_pts; seed=seed+1)
        X_d   = rs_design_matrix(pts_d)
        push!(cond_dopt, cond(X_d' * X_d))
    end
    return cond_random, cond_dopt
end

n_vars = 3
ns_min = min_rs_points(n_vars)
n_pts_range = ns_min:(ns_min + 30)

cond_r, cond_d = condition_number_study(n_vars, n_pts_range)

plot(n_pts_range, cond_r, lw=2, color=:crimson, label="Random placement",
     xlabel="Sample size NS", ylabel="Condition number κ(X'X)",
     title="Design Matrix Conditioning (n = $n_vars variables)",
     yscale=:log10, legend=:topright, grid=true, gridalpha=0.3)
plot!(n_pts_range, cond_d, lw=2, color=:steelblue, label="D-Optimal placement")

model — a *response surface* — that approximates the relationship between a set of

context of BLISS-RS, the factors are the system-level design variables $Z$ and

the same output. This changes how sample points should be placed — classical designs

## Construction of Empirical Models

### Linear Regression

In most RSM applications, the true response function is unknown and we seek a
polynomial approximation. The simplest choice is a **first-order (linear) model**
with $n$ independent variables:

$$
Y = \beta_0 + \sum_{i=1}^{n} \beta_i x_i + \varepsilon
$$ {#eq-linear-model}

where $\beta_0, \beta_1, \ldots, \beta_n$ are unknown regression coefficients and
$\varepsilon$ is a random error (zero for deterministic computer experiments).

This is a *linear model* because it is a linear function of the unknown parameters
$\beta$, not necessarily of the variables $x_i$. Interaction terms ($\beta_{12} x_1
x_2$) and squared terms ($\beta_{11} x_1^2$) can be added while maintaining
linearity in parameters.

Suppose the response $Y$ is observed $m$ times. Each observation $y_i$ satisfies:

$$
y_i = \beta_0 + \sum_{j=1}^n \beta_j x_{ij} + \varepsilon_i, \quad i = 1, 2, \ldots, m
$$ {#eq-obs}

In matrix form:

$$
\{y\} = [X]\{\beta\} + \{\varepsilon\}
$$ {#eq-matrix-form}

where $\{y\}$ is $m \times 1$, $[X]$ is an $m \times p$ *design matrix*, $\{\beta\}$
is $p \times 1$, and $\{\varepsilon\}$ is $m \times 1$.

**Least squares estimation** minimises the sum of squared residuals:

$$
L = \{\varepsilon\}^T\{\varepsilon\} = (\{y\} - [X]\{\beta\})^T(\{y\} - [X]\{\beta\})
$$ {#eq-ls}

Setting $\partial L / \partial \{\beta\} = 0$ yields the *normal equations*:

$$
[X]^T[X]\{b\} = [X]^T\{y\}
$$ {#eq-normal}

The least-squares estimator $\{b\}$ of $\{\beta\}$ is:

$$
\{b\} = \left([X]^T[X]\right)^{-1}[X]^T\{y\}
$$ {#eq-ls-solution}

The fitted values are:

$$
\{\hat{y}\} = [X]\{b\}
$$ {#eq-fitted}

### Second-Order Models

For BLISS-RS, first-order models are generally insufficient because the BB
responses exhibit curvature. A **second-order (quadratic) model** adds pure
quadratic and cross-product (interaction) terms:

$$

  + \sum_{i < j} \beta_{ij} x_i x_j
$$ {#eq-quadratic}

The minimum number of observations required to uniquely fit a second-order model
in $n$ variables is:

$$
N_S = \frac{n^2 + 3n + 2}{2}
$$ {#eq-ns-second-order}

### Julia Implementation

```{julia}
#| label: lst-rs-functions
#| code-fold: true

# Build the [X] design matrix for a second-order model
# (duplicated here for clarity; full implementation in src/bliss_rs.jl)
function build_quadratic_X(pts::Matrix{Float64})
    m, n = size(pts)
    p = 1 + n + n*(n+1)÷2
    X = ones(m, p)
    X[:, 2:n+1] = pts
    col = n + 2
    for i in 1:n, j in i:n
        X[:, col] = pts[:, i] .* pts[:, j]
        col += 1
    end
    return X
end

# Demonstrate on a simple 2-variable quadratic function
# f(x1, x2) = 3 + 2x1 - x2 + 0.5*x1^2 + x1*x2 - 0.3*x2^2
true_β = [3.0, 2.0, -1.0, 0.5, 1.0, -0.3]
f_true(x1, x2) = true_β' * [1, x1, x2, x1^2, x1*x2, x2^2]

# Sample 12 points (> NS = 6 for n=2)
rng = MersenneTwister(42)
pts = 2 .* rand(rng, 12, 2) .- 1.0    # in [-1, 1]²
y   = [f_true(pts[i,1], pts[i,2]) for i in 1:12]

# Fit the RS
X_mat = build_quadratic_X(pts)
b_hat = (X_mat' * X_mat) \ (X_mat' * y)

@printf("True β:   [%.2f, %.2f, %.2f, %.2f, %.2f, %.2f]\n", true_β...)
@printf("Fitted b: [%.2f, %.2f, %.2f, %.2f, %.2f, %.2f]\n", b_hat...)
@printf("Max coefficient error: %.4f\n", maximum(abs.(b_hat .- true_β)))
```

With 12 points for a 2-variable model ($N_S = 6$), the least-squares fit recovers
the true quadratic exactly (within floating-point precision), confirming the
implementation.

## Error Analysis of Fitted Response Surfaces

### Test Functions

Altus (2002) assessed RS accuracy using two test functions across a range of
dimensionalities $N_X$ and sample sizes $N_S$ [@altus2002blissrs, §4.2]:

**Test 1** — Fitting a quadratic RS to a *quadratic* function. A correctly
implemented quadratic model recovers a quadratic function exactly when $N_S \geq
N_S^{\min}$, so this is a code-verification test, not a fidelity test.

**Test 2** — Fitting a quadratic RS to the generalised **Rosenbrock (Banana)
function**, which has a narrow curved valley and is substantially non-quadratic:

$$
f(\mathbf{x}) = \sum_{i=1}^{N_X - 1}\left[100(x_{i+1} - x_i^2)^2 + (x_i - 1)^2\right]
$$ {#eq-rosenbrock}

### Corrected Error Metrics

The original thesis reported a single point-wise normalized error:

$$
\text{Error}_{\text{Altus}} = \frac{|f_{\text{actual}} - f_{RS}|}{|f_{\text{actual}}|}
$$ {#eq-error-metric-original}

This metric has two deficiencies identified during IV&V:

1. **Singularity** — when $f_{\text{actual}} \to 0$ (e.g. the aeroelastic twist
   variable $\Theta$, which can change sign during optimization), the metric
   diverges. The original MATLAB code masked this with an implicit numerical floor.

2. **Point-wise only** — no summary statistic across the design space, no
   out-of-sample validation, no confidence information.

The corrected validation framework uses three complementary metrics:

$$
\text{RMSE} = \sqrt{\frac{1}{m}\sum_{i=1}^{m}(y_i - \hat{y}_i)^2}
$$ {#eq-rmse}

$$
R^2 = 1 - \frac{\sum(y_i - \hat{y}_i)^2}{\sum(y_i - \bar{y})^2}
$$ {#eq-r2}

$$
\text{NMAE} = \frac{1}{m}\sum_{i=1}^{m}
  \frac{|y_i - \hat{y}_i|}{\max(|y_i|,\, \varepsilon \cdot \max_j|y_j|)}
$$ {#eq-nmae}

where $\varepsilon = 10^{-8}$ provides the stabilizing floor absent from Eq. 16.
All three are computed both **in-sample** (fit quality) and **leave-one-out
cross-validated** (predictive quality). LOOCV is the standard for deterministic
computer experiments because there is no measurement noise to justify a held-out
test set.

    plot!(ns_vec, max.(rmse_vec, 1e-4), lw=2, color=col,
          label="NX = $nv  (NS_min = $ns_min)")
    vline!([ns_min], color=col, lw=1, linestyle=:dot, label="")
end
p
```

@fig-error-analysis reproduces and extends the accuracy study from Altus (2002)
Figs. 13–16 using LOOCV RMSE in place of the point-wise metric. The key finding
is unchanged: for low-dimensional BBs ($N_X \leq 5$), a quadratic RS achieves
acceptable accuracy with moderate over-determination ($N_S \approx 1.5 N_S^{\min}$).
For high-dimensional BBs ($N_X = 8$, as in the aerodynamics BB), predictive accuracy
remains poor until $N_S \gg N_S^{\min}$, motivating either additional sample points
or a more expressive surrogate (Gaussian Process, Chapter 9).

### Response Surface Visualization

#| label: fig-rs-surface

#| code-fold: true

rng = MersenneTwister(77)
n_train = 20

Z_true = [rosenbrock([xi, yi]) for yi in yg, xi in xg]

p1 = contourf(xg, yg, min.(Z_true, 100), color=:viridis, levels=20,
              title="True Rosenbrock", xlabel=L"x_1", ylabel=L"x_2",
              colorbar_title="f(x)", clim=(0,100))

p2 = contourf(xg, yg, clamp.(Z_rs, 0, 100), color=:viridis, levels=20,
              title="Quadratic RS Approximation", xlabel=L"x_1", ylabel=L"x_2",
              colorbar_title="f̂(x)", clim=(0,100))

plot(p1, p2, size=(800, 350))
```

The contour plots in @fig-rs-surface illustrate an important limitation of quadratic
surrogates: the actual Rosenbrock valley follows a parabolic curve $x_2 = x_1^2$,
while the quadratic RS can only represent it as an ellipsoidal bowl. This mismatch
motivates the *interval reduction* strategy in Chapter 6 — by shrinking the design
space region around the optimum each iteration, subsequent RS models are fitted to
smaller, more nearly-quadratic sub-regions where the approximation is more accurate.

## Conditioning and Over-Determination

The normal equations (@eq-normal) can become ill-conditioned when sample points are
clustered or collinear. Good conditioning requires the *design matrix* $[X]$ to have
columns that are mutually uncorrelated — a property promoted by the D-optimal design
strategy described in Chapter 5.

```{julia}
#| label: fig-conditioning
#| fig-cap: "Condition number of X'X as a function of sample size and point placement strategy. Clustered random points can produce high condition numbers, increasing sensitivity to numerical errors. Well-placed D-optimal designs maintain low condition numbers."
#| code-fold: true

function condition_number_study(n_vars::Int, n_pts_range, n_trials::Int=10; seed=42)
    rng = MersenneTwister(seed)
    cond_random = Float64[]
    cond_dopt   = Float64[]

    for n_pts in n_pts_range
        # Random points
        conds = Float64[]
        for trial in 1:n_trials
            pts = rand(rng, n_pts, n_vars)
            X   = rs_design_matrix(pts)
            push!(conds, cond(X' * X))
        end
        push!(cond_random, median(conds))

        # D-optimal points
        pts_d = d_optimal_points(n_vars, n_pts; seed=seed+1)
        X_d   = rs_design_matrix(pts_d)
        push!(cond_dopt, cond(X_d' * X_d))
    end
    return cond_random, cond_dopt
end

n_vars = 3
ns_min = min_rs_points(n_vars)
n_pts_range = ns_min:(ns_min + 30)

cond_r, cond_d = condition_number_study(n_vars, n_pts_range)

plot(n_pts_range, cond_r, lw=2, color=:crimson, label="Random placement",
     xlabel="Sample size NS", ylabel="Condition number κ(X'X)",
     title="Design Matrix Conditioning (n = $n_vars variables)",
     yscale=:log10, legend=:topright, grid=true, gridalpha=0.3)
plot!(n_pts_range, cond_d, lw=2, color=:steelblue, label="D-Optimal placement")
hline!([1e6], color=:black, linestyle=:dash, lw=1, label="Warning threshold")
```
